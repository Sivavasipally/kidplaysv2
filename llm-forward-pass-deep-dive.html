Step 00 — The shape of the journey

One prompt, nine transformations.

An LLM does not "understand" text the way you do. It runs a fixed sequence of matrix operations — the forward pass — that turns a string into a vector of probabilities over its entire vocabulary. The word it predicts becomes the next token, and the loop repeats. Everything below is one trip through that machine.

We will trace the journey on a concrete configuration — the dimensions of a GPT-2-class model — so every shape is a real number you can hold onto:

A note on "agents." An LLM agent (the kind you orchestrate in Dify or via MCP) is this forward pass wrapped in a loop with tools, memory, and a controller. The model itself only ever does what follows — predict the next token. The "agentic" intelligence lives in how you route those predictions back into actions. Understanding the box above is understanding the engine inside every agent.
Step 01 — Tokenization

Text becomes integers.

The model has no notion of letters or words. Before anything else, the tokenizer chops the string into tokens — sub-word chunks drawn from a fixed vocabulary of 50,257 entries — and replaces each with its integer ID. GPT-style models use Byte-Pair Encoding (BPE): common words become a single token, rarer words split into pieces, and a leading space is glued onto the token that follows it (shown here as ·).

Our prompt tokenizes to roughly the sequence below. (IDs are illustrative of GPT-2's BPE table; the exact split depends on the tokenizer.)

Class9487 ify1958 ·the262 ·text2420 ·into656 ·neutral8500 ,11 ·negative4633 ,11 ·or393 ·positive3967 .13 ·Text8255 :25 ·I314 ·think892 ·the262 ·vacation14600 ·is318 ·okay8788 .13 ·Sent11352 iment3681 :25 ·220

That is T = 25 tokens. The model's entire job is to predict the 26th — the token that comes after the trailing space following Sentiment:. The output of this step is a plain list of 25 integers:

token id sequence ids = [9487, 1958, 262, 2420, 656, 8500, 11, 4633, 11, 393, 3967, 13, 8255, 25, 314, 892, 262, 14600, 318, 8788, 13, 11352, 3681, 25, 220]
shape: (T,) = (25,) — just integers, no meaning attached yet
Why sub-words? A pure word vocabulary can't handle typos, new words, or "vacationing." A pure character vocabulary makes sequences painfully long. BPE is the compromise: ~50k pieces that recombine to spell anything, keeping common words cheap and rare ones merely a few tokens.
Step 02 — Token embeddings

Integers become vectors.

An integer like 8788 carries no relationships — 8788 isn't "more positive" than 4633. So each ID is used to look up a row in the embedding matrix E, a learned table with one 768-dimensional vector per vocabulary entry. This is a pure lookup: no multiplication, just indexing.

embedding matrix E ∈ ℝ50257 × 768 — one learned row per token

ei = E[ ids[i] ] — grab the row for token i

E[8788] = [ 0.021, −0.114, 0.318, … , 0.077 ] (768 numbers, "okay")

Stacking the 25 looked-up vectors produces the first real tensor in the network:

embedded input X = stack(e0, e1, …, e24)
X ∈ ℝ25 × 768 — 25 token vectors, each 768-dimensional
These vectors already mean something. Because E is learned during training, words used in similar contexts end up near each other in this 768-d space. "okay," "fine," and "alright" sit close together; "vacation" lives near "holiday" and "trip." This is the raw material attention will mix.
Step 03 — Positional information

Telling the model where each token sits.

There is a problem. The attention mechanism (next section) treats its input as a set — it has no built-in sense of order. Yet "vacation is okay" and "okay is vacation" must differ. The fix: add a positional vector pi (also 768-d) to each token embedding, encoding its index in the sequence.

position-aware input xi = ei + pi — elementwise add, shape unchanged

X0 ∈ ℝ25 × 768 — still 25 × 768, now order-aware

Classic transformers build pi from sines and cosines of different frequencies, so each position gets a unique fingerprint the model can decode:

PE(pos, 2k) = sin( pos / 100002k/768 )
PE(pos, 2k+1) = cos( pos / 100002k/768 )
low dimensions oscillate fast (fine position), high dimensions slowly (coarse position)
Modern variants. Many current models replace this with learned position embeddings or rotary embeddings (RoPE), which rotate Q and K by an angle proportional to position. The principle is identical: inject order so attention can use it. From here on we carry one matrix, X₀ ∈ ℝ²⁵ˣ⁷⁶⁸, into the stack of transformer blocks.
Step 04 — Self-attention · the core

Attention is all you need.

This is the engine. Each token asks every other token a question, reads their answers, and updates itself with a weighted blend of what it found relevant. The mechanism is built from three projections of the input — Query, Key, and Value.

Q — Query: "what am I looking for?" K — Key: "what do I offer?" V — Value: "what do I contribute?"

4.1 — Project X into Q, K, V

Three learned weight matrices turn each token vector into three new vectors. Within a single attention head these project down to 64 dimensions (768 ÷ 12 heads).

linear projections (per head) Q = X0 WQ WQ ∈ ℝ⁷⁶⁸ˣ⁶⁴ → Q ∈ ℝ²⁵ˣ⁶⁴
K = X0 WK WK ∈ ℝ⁷⁶⁸ˣ⁶⁴ → K ∈ ℝ²⁵ˣ⁶⁴
V = X0 WV WV ∈ ℝ⁷⁶⁸ˣ⁶⁴ → V ∈ ℝ²⁵ˣ⁶⁴

4.2 — Score: how much should each token attend to each other?

Take the dot product of every query with every key. A large dot product means "this key is relevant to my query." Dividing by √dhead = √64 = 8 keeps the numbers from exploding (which would make the upcoming softmax too spiky).

scaled scores S = Q Kᵀ / √64
(25 × 64) · (64 × 25) = 25 × 25 — one score for every (query token, key token) pair

4.3 — Causal mask: no peeking ahead

Because the model generates left-to-right, token i may only attend to tokens ≤ i. Future positions are set to −∞ before the softmax, so they receive zero weight. This is why the last token — the one we predict from — can see the entire prompt, but not vice-versa.

apply causal mask Sij ← −∞ if j > i — upper triangle blanked out

4.4 — Softmax: turn scores into a distribution

Each row of S is squashed into positive weights that sum to 1. Now every token has a recipe: "I am X% token A, Y% token B…". Here is the actual softmax, applied row-wise:

attention weights Aij = exp(Sij) / Σk≤i exp(Sik)
A ∈ ℝ²⁵ˣ²⁵, every row sums to exactly 1.0

Let's make it concrete. Consider the last token's row — the position that decides the sentiment. Suppose its query, dotted against four key tokens, yields these scaled scores:

key token·okay·Sentiment:·vacation·think
score Sᵢⱼ4.03.22.51.0
exp(Sᵢⱼ)54.6024.5312.182.72
weight Aᵢⱼ0.5810.2610.1300.029

The model concentrates 58% of its attention on "okay" and 26% on the Sentiment: cue — exactly the two tokens that determine the answer. The sum of all weights in the row is 1.000.

4.5 — Weighted sum: gather the values

Finally, multiply the weights by V. Each token's output is the attention-weighted average of all value vectors — it has now "read" the rest of the sequence.

attention output Attention(Q,K,V) = softmax( QKᵀ / √dk ) V
(25 × 25) · (25 × 64) = 25 × 64 — one enriched vector per token
Step 05 — Multi-head attention

Twelve perspectives at once.

One head learns one kind of relationship — maybe "adjective → noun." To capture syntax, coreference, sentiment cues and more simultaneously, the model runs 12 independent heads in parallel, each with its own WQ, WK, WV projecting into a separate 64-d subspace. Their outputs are concatenated back to 768 and mixed by an output projection WO.

combine heads headh = Attention(X WQh, X WKh, X WVh) → 25 × 64

MultiHead = concat(head1, …, head12) WO
concat → 25 × 768, then × WO ∈ ℝ⁷⁶⁸ˣ⁷⁶⁸ → 25 × 768
Step 06 — The transformer block

Attention, then thinking.

Attention moves information between tokens. The feed-forward network (FFN) then processes each token independently — expanding to 3,072 dimensions, applying a non-linearity, and projecting back. Two glue components hold it together: residual connections (add the input back, so gradients flow and information isn't lost) and layer normalization (re-center and re-scale each vector for stable training).

one block (pre-norm form) X′ = X + MultiHead( LayerNorm(X) ) — attention sub-layer + residual
X″ = X′ + FFN( LayerNorm(X′) ) — FFN sub-layer + residual
feed-forward network FFN(x) = GELU( x W1 + b1 ) W2 + b2
W1 ∈ ℝ⁷⁶⁸ˣ³⁰⁷² (expand) · W2 ∈ ℝ³⁰⁷²ˣ⁷⁶⁸ (contract)
Step 07 — Depth

Stack the block twelve times.

Because a block's output has the same shape as its input (25×768), blocks compose. The sequence passes through all 12, and the representations grow richer with depth: early layers track surface features (parts of speech, local phrases), middle layers assemble meaning, and late layers sharpen toward the specific prediction. By the final block, the last token's vector is a 768-d summary of "given everything above, what comes next?"

the full trunk X0 → block₁ → block₂ → … → block₁₂ → Xfinal
Xfinal ∈ ℝ²⁵ˣ⁷⁶⁸ — we only need its last row, Xfinal[24]
Parameter count, for intuition. Each block holds ~7M parameters (attention + FFN); ×12 blocks + embeddings ≈ 124M total for GPT-2 small. A frontier model swaps these numbers for billions — wider dmodel, more heads, far more layers — but the operations on this page are exactly the same.
Step 08 — From vector to vocabulary

Scoring every possible next word.

We now take just the last token's final vector and project it back to vocabulary space using the transpose of the embedding matrix (weight tying). This produces one logit — an unnormalized score — for all 50,257 possible next tokens.

unembedding → logits z = Xfinal[24] · Eᵀ (1 × 768) · (768 × 50257) = 1 × 50257
z is a score for every token; higher = "more likely to come next"

A final softmax turns those 50,257 logits into a probability distribution. We only care about the three candidate tokens the instruction allowed — ·neutral, ·positive, ·negative — whose logits dominate because the prompt steered the model there. Suppose the network produced:

candidate tokenlogit zexp(z)softmax probability
·neutral7.92697.30.689 (68.9%)
·positive7.11211.970.309 (30.9%)
·negative2.07.390.002 (0.2%)
final softmax P(tokent) = exp(zt / τ) / Σv exp(zv / τ)
τ = temperature. τ→0 makes it greedy (always the top token); τ>1 flattens toward random.

With greedy decoding the model emits ·neutral — the right call for "I think the vacation is okay," a lukewarm verdict the model correctly reads as more neutral than positive. That token is appended, and if generation continued the whole 26-token sequence would flow back to Step 02. Here, the answer is complete.

Step 09 — Where the platform sits

From a token to an agent.

Everything above is one forward pass — pure prediction, no will of its own. An orchestration layer (Dify, an MCP host, a Spring-Boot service calling the API) is what turns this into an agent: it builds the prompt, sends it through the forward pass, reads the output token(s), and decides what to do next — call a tool, query a RAG store, hand off to another step, or return the answer.

The one-sentence summary

A prompt is split into tokens, each token becomes a 768-d learned vector, attention lets every token read every earlier token, twelve blocks refine those vectors, and the last vector is projected to a probability over 50,257 words — from which the next word is drawn. Repeat, and you have language. Wrap it in tools, and you have an agent.

"Attention lets each word decide, for itself, which other words matter."
