+ — The internals beneath the animation

Six mechanisms worth knowing.

The animations show the flow. These cards explain why each piece is shaped the way it is — the conceptual machinery that the diagrams gloss over.

the highway
Residual stream

Every block adds its output back to its input rather than replacing it: x ← x + sublayer(x). So the 768-d vector is a running ledger that each layer reads from and writes to. Information from layer 1 can survive untouched all the way to layer 12 — and gradients flow straight back during training, which is what makes 12+ layers trainable at all.

why ÷√dₖ
The scaling factor

Dot products of two 64-d vectors have variance ∝ 64. Left unscaled, scores would be huge, the softmax would saturate to a near one-hot spike, and gradients would vanish. Dividing by √64 = 8 keeps the score variance ≈ 1, so the softmax stays soft and the model can actually learn nuanced attention.

memory, not math
FFN as key–value store

The feed-forward block — 768→3072→768 with GELU — behaves like a learned lookup table. The first matrix's 3,072 rows act as keys that detect patterns ("is this about sentiment?"); the second matrix's columns are values written into the residual stream when those keys fire. Most of a model's facts live here, not in attention.

geometry
LayerNorm

Before each sublayer, the vector is re-centered to mean 0 and scaled to unit variance, then stretched by learned gain/bias. Geometrically it projects every token onto a sphere of fixed radius — removing "how loud" a vector is so the layer attends only to "which direction," i.e. what it means, keeping activations stable across depth.

division of labor
Head specialization

The 12 heads aren't redundant. Trained networks reliably grow specialists: a previous-token head, a syntax head linking verbs to subjects, an induction head that copies repeated patterns, a head that tracks the Sentiment: cue. Twelve cheap 64-d views beat one expensive 768-d one.

the loop
Autoregression & the KV cache

The model predicts one token, appends it, and re-runs. Naively that recomputes everything; in practice the Keys and Values of past tokens are cached, so each new token only computes its own Q/K/V and attends over the stored cache. That cache is why long contexts eat memory — it grows linearly with every token generated.

Tying it back. When you tune a prompt — adding a role, fencing input in delimiters, listing exact labels — you're nudging this exact machinery: shifting which heads fire, which FFN keys activate, and where the final softmax puts its mass. Prompt engineering is, mechanically, steering the forward pass you just watched.
